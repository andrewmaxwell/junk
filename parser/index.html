Notes:

  • Each line of the grammar is a "type", and each type has a name followed by a colon followed by either a regex or a list of types/literals.
  • The grammar must contain a "main" type.
  • The order of types does not matter.
  • Regular expressions must start with ^ and should probably end with a \s* to ignore whitespace after them.
  • Non-Regex types are made of one or more parts concatenated by single spaces.
  • The parts themselves can be made optional by adding a question mark.
  • The parts can also match multiple other types by concatenating them with pipes.
  • If a part does not match a type, it is treated as a literal.
  • White space is ignored after literals (but not regexes).
  • The grammar is case sensitive. I could add a control for making it case-insensitive if necessary.
